# 병합 규칙 (Merge Rules)

## 병합 단계

### 1. JSON 로드

`/tmp/context-sync/sources/` 디렉토리에서 존재하는 모든 JSON 파일을 로드한다.
파싱 실패한 파일은 건너뛰고 에러를 기록한다.

### 2. 하이라이트 추출

다음 조건 중 하나 이상 충족하는 항목을 하이라이트로 추출:

| 조건 | 우선순위 |
|------|---------|
| priority == "high" | 자동 포함 |
| action_required == true | 자동 포함 |
| 여러 소스에서 동일 주제 언급 | 우선순위 상향 |

### 3. 소스별 상세 정리

각 소스별로 items를 시간순(최신 먼저)으로 정렬:
- 항목당 최대 3줄 요약
- 소스별 stats 요약 포함

### 4. 액션 아이템 통합

모든 소스에서 `action_required == true`인 항목을 추출하여 단일 목록으로 병합:
- 시간순 정렬 (긴급한 것 먼저)
- 중복 제거 (같은 주제를 여러 소스에서 언급한 경우)
- 체크박스 형식으로 출력

### 5. 타임라인 생성

모든 소스의 items를 시간순으로 합쳐 타임라인을 생성:
- 최대 20개 항목
- 각 항목에 소스 태그 표시: `[slack]`, `[gmail]`, `[cal]`, `[gh]`, `[work]`
- 시간 형식: HH:MM

## 중복 제거 규칙

| 상황 | 처리 |
|------|------|
| Slack + Gmail에서 같은 미팅 언급 | Calendar 항목 유지, 나머지 병합 |
| GitHub PR 알림 + Slack 멘션 | GitHub 항목에 Slack 맥락 추가 |
| 같은 이벤트의 다른 관점 | 가장 상세한 항목 유지 |

## 소스 없음 처리

활성 소스가 모두 데이터를 반환하지 못한 경우:
- "수집 가능한 데이터가 없습니다" 메시지 출력
- 각 소스의 실패 사유 표시
